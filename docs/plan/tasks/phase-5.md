# Task specs — phase 5 (figures)

Lane key and **standing constraints block** as in `phase-1.md` — copy the
block into every delegation. Lane tags state routing intent; when a tagged
lane is unavailable the executing lane and the substitution are recorded in
the task report (orchestration doctrine).

Stack, per `../../research/rust-ecosystem.md` §7: plotters (+plotters-svg)
for rendering, svg2pdf for PDF, hand-templated PGFPlots and Typst/CeTZ text
emission, Vega JSON emission without any V8 dependency, generated
matplotlib/R/Julia scripts alongside exported data.

Committed APIs the figure model consumes (verified in-tree):
`phx_spectrogram::{Tile, Slice}` (raw dB + axes),
`phx_pitch::{PitchTrack, TimeSpan}` (frames + span statistics),
`phx_formant::{FormantTrack, FormantFrame, FormantPoint}`,
`phx_intensity::IntensityTrack` (`iter() → (time, db)`),
`phx_annot::{Annotation, TierSlot, Tier}`,
`phx_render::{Colormap, DisplayMapping, Theme, colorize}`.

**Formant caveat (from `phase-4.md`):** T2.6 closed accept-with-documentation;
`track_smoothed` weights remain implementation choices. Figure layers built
from formants record in the figure model whether they carry raw or smoothed
tracks, and every export marks smoothed tracks as such in the figure's
caption metadata until the weights graduate.

### T5.1 · codex · figure model (phx-figure core)
**Objective.** A self-contained, backend-agnostic figure description: the
single input type every exporter consumes.
**Files.** `crates/phx-figure/src/{model,builder,style}.rs`.
**Interfaces.**
```rust
pub struct Figure { pub size: SizeSpec /* cm|in|pt, physical */, pub theme: Theme,
  pub panels: Vec<Panel>, pub caption_meta: CaptionMeta /* params provenance, track kind */ }
pub struct Panel { pub layers: Vec<Layer>, pub time_axis: Axis, pub value_axis: Axis,
  pub height_share: f64 }
pub enum Layer {                       // all data EMBEDDED, no AudioId/engine refs
  Waveform { minmax: Vec<MinMax>, span: TimeSpan, style: LineStyle },
  Spectrogram { db: Vec<f32>, width: u32, height: u32, t: [f64; 2], f: [f64; 2],
                display: DisplayMapping, colormap: Colormap },   // raw dB, colorized per backend
  PitchLine { points: Vec<(f64, f64)>, unit: PitchUnit, style: LineStyle },
  FormantSpeckle { frames: Vec<SpeckleFrame>, smoothed: bool, style: SpeckleStyle },
  IntensityLine { points: Vec<(f64, f64)>, style: LineStyle },
  Tiers { tiers: Vec<TierData>, style: TierStyle },
  SpectralSlice { bins: Vec<(f64, f64)>, style: LineStyle },
}
pub struct FigureBuilder { /* from_view(…): assembles Layers out of the committed
  track/tile/annotation types listed above; all conversions live here */ }
```
**Constraints.** Standing set. serde on everything (the model is also the
dialog↔worker wire format). Physical units resolved by the model
(`px_at(dpi)`); no backend does unit math. The spectrogram layer stores raw
dB so backends re-colorize per theme; `colorize` is called at export time,
never at build time.
**Verification.** Builder tests from fixture-derived tracks (each committed
type → expected layer contents); serde round-trip property test; a
`reference_figure()` constructor (waveform + spectrogram + pitch + one tier
from `arctic_bdl_a0001`) that every backend task reuses as its gate input.

### T5.2 · grok · raster/vector backends: SVG, PNG, PDF
**Depends on T5.1.**
**Objective.** Pixel-and-vector-faithful rendering of `Figure`.
**Files.** `crates/phx-figure/src/backends/{svg,png,pdf}.rs`.
**Interfaces.** `pub fn to_svg(fig: &Figure) -> String`,
`to_png(fig: &Figure, dpi: f64) -> Vec<u8>`,
`to_pdf(fig: &Figure) -> Vec<u8>` (svg2pdf over `to_svg`).
**Requirements.** plotters/plotters-svg drawing; the spectrogram layer
embeds as a base64 PNG `<image>` (colorized at export DPI) while axes,
tracks, tiers, and text stay vector; fonts: a bundled permissively licensed
text face with system-font fallback documented; light and dark themes both
render (dark for screen, light default for print).
**Constraints.** Standing set. SVG is the source of truth: PNG and PDF
derive from the same scene graph; no backend re-implements layout. svg2pdf
and plotters versions go through `[workspace.dependencies]`.
**Verification.** `reference_figure()` exported to all three; SVG parses
(usvg) and contains one `<image>` per spectrogram layer + vector paths for
tracks; PNG at 2× DPI pixel-diffs against a rasterized SVG within 1%
differing pixels; PDF opens under `pdftoppm` and visually diffs the same
way. Golden files regenerated by a committed script only.

### T5.3 · codex · text-emission backends: TikZ, Typst, Vega, code+data, GraphML
**Depends on T5.1. Parallel to T5.2 (disjoint modules).**
**Objective.** Publication-source exports that humans keep editing.
**Files.** `crates/phx-figure/src/backends/{tikz,typst,vega,code,graphml}.rs`.
**Interfaces.** `to_tikz(fig) -> TextExport` (PGFPlots; spectrogram as a
sidecar PNG referenced via `\addplot graphics`), `to_typst(fig) ->
TextExport` (CeTZ canvas; spectrogram via `image()` sidecar),
`to_vega(fig) -> String` (Vega-Lite v5 JSON; tracks as inline data,
spectrogram as a data-URI image mark), `to_code(fig, lang: CodeLang) ->
CodeExport /* {script, data_files: CSV per layer + dB matrix} */` for
`Python | R | Julia`, `to_graphml(annotation_layers) -> String` (nodes =
intervals/points with tier + label + span attributes; edges = precedence
within tier and alignment/containment across tiers).
`TextExport`/`CodeExport` carry a main file plus named sidecars; the UI
writes them as a bundle.
**Constraints.** Standing set. Generated sources are readable artifacts:
sensible variable names, a header comment stating only what the file is and
its axes/units, no generator narration or session wording (the de-ai file
rules apply to generated text verbatim). matplotlib code uses no seaborn;
R uses base ggplot2; Julia uses Makie. Vega output validates against the
Vega-Lite v5 JSON schema fetched into `tests/` once, offline afterwards.
**Verification.** For `reference_figure()`: TikZ compiles under `latexmk
-pdf` in the toolchain container (T5.5); Typst compiles under `typst
compile`; each generated script runs (python/Rscript/julia) writing a PNG
that visually matches the T5.2 PNG within coarse tolerance (5% differing
pixels at 100 dpi — font differences expected); Vega JSON passes schema
validation; GraphML loads in a `graphml` parser test and node/edge counts
match the annotation.

### T5.4 · grok · engine surface + export dialog
**Depends on T5.1; UI portion also on T1.6 stack (merged).**
**Objective.** `ux.md` §Figure-export dialog with a live theme-aware
preview; wow-moment 5.
**Files.** `crates/phx-engine` (`build_figure(id, view: &ViewSpec, sel:
Option<Selection>) -> Figure`, `export_figure(fig, format) -> ExportBundle`),
`phx-wasm` plumbing, `apps/ui/src/lib/` (ExportDialog, FigurePreview,
LayerToggles), `apps/web` wiring.
**Requirements.** Dialog assembles the figure from the current view or
selection with per-layer toggles (waveform / spectrogram / pitch / formants /
intensity / tiers), size in cm/in with live reflow, palette switch including
grayscale-for-print, theme preview toggle (the preview re-renders through
`to_svg` in the worker — the preview IS the SVG backend, so preview = export
by construction); smoothed formants labeled per the caveat above; export
downloads the bundle (main file + sidecars zipped when more than one).
**Constraints.** Standing set. Preview renders off the UI thread; dialog is
an overlay, never a browser modal; both themes screenshot-verified.
**Verification.** Playwright: open dialog on a selection, toggle layers,
switch viridis→grayscale, flip preview theme, export SVG and PDF, assert
the exported SVG equals the previewed SVG byte-for-byte; screenshots
light+dark of the dialog.

### T5.5 · sonnet · toolchain verification harness
**Objective.** CI-side proof that text exports actually compile and run.
**Files.** `tools/figcheck/` (containerized: TeX Live small + latexmk, typst,
python+matplotlib, R+ggplot2, julia+Makie; a runner script per backend),
CI job `figures` (runs on changes under `crates/phx-figure/`).
**Constraints.** Standing set; container pinned by digest; job skips with an
explicit "skipped" status where the container is unavailable, never silently
passes.
**Verification.** CI run green with all five checks executed; a deliberately
broken TikZ template fails the job.

### T5.6 · architect · phase gate review
Read diffs; run figcheck; compare exported PDF/SVG against the on-screen
preview screenshots; verify the caption metadata records analysis parameters
(provenance rule); close against roadmap.md phase-5 gate.

## Sequencing

T5.1 first — it is the contract everything else compiles against. T5.2,
T5.3, and T5.4's engine half then run in parallel (disjoint files); T5.4's
dialog needs T5.2's SVG backend for its preview. T5.5 can start immediately
(container + runner scripts need no figure code) and blocks only T5.6.
GraphML has no upstream consumer yet; if it slips, it moves to backlog
without blocking the gate — the roadmap gate names the other backends.
