# Task specs — phases 0 and 1

Every implementation task inherits the standing constraints below; specs
handed out for implementation must copy them in.

**Standing constraints (copy into every delegation):**
- License MIT OR Apache-2.0. Clean-room: implement from the citations given
  in the spec (papers, Praat public manual pages); never read or port Praat
  source, parselmouth source, or any GPL code (including praatfan-core-rs,
  aubio). If a needed detail is missing from the cited sources, stop and
  report the gap instead of improvising or peeking.
- Rust edition 2024; `cargo fmt` clean; `cargo clippy --workspace -- -D
  warnings` clean; every public item doc-commented; parameter structs derive
  `Default` matching the documented Praat defaults with provenance in the doc
  comment.
- Conventional Commits, one logical change per commit, no AI attribution of
  any kind, no tool/agent names in branches.
- Code comments state constraints the code cannot show; no narration, no
  references to this task or session.

## Phase 1

### T1.1 · phx-audio
**Objective.** Audio container + WAV I/O + resampling.
**Files.** `crates/phx-audio/src/*`.
**Interfaces.**
```rust
pub struct Audio { /* planar f32 channels, sample_rate: f64, name: Option<String> */ }
impl Audio {
  pub fn from_wav_bytes(bytes: &[u8]) -> Result<Self, AudioError>;
  pub fn to_wav_bytes(&self, bits: BitDepth) -> Result<Vec<u8>, AudioError>;
  pub fn channel(&self, i: usize) -> &[f32];
  pub fn mono_mix(&self) -> Cow<'_, [f32]>;
  pub fn duration(&self) -> f64;
  pub fn resampled(&self, target_hz: f64, quality: ResampleQuality) -> Result<Self, AudioError>;
  pub fn slice_samples(&self, range: Range<usize>) -> AudioView<'_>;
}
```
`AudioView` borrows without copying. Symphonia decode behind a
`decode-extended` feature (desktop only), hound always.
**Constraints.** Standing set. rubato for resampling (windowed-sinc preset);
integer WAV formats normalized to [-1, 1] f32; reject > 2 GB allocations with
a typed error.
**Verification.** Round-trip tests (16/24/32-bit, mono/stereo); resample a
440 Hz tone 44.1k→16k and assert peak frequency within 0.1 Hz via FFT; all
tests pass under `--target wasm32-unknown-unknown` compile check.

### T1.2 · phx-dsp
**Objective.** Shared DSP primitives with an absolute-time frame grid.
**Files.** `crates/phx-dsp/src/*`.
**Interfaces.**
```rust
pub enum Window { Hanning, Gaussian { effective_len_factor: f64 }, Kaiser { beta: f64 } }
pub fn window_samples(w: Window, n: usize) -> Vec<f64>;
pub struct FrameGrid { /* derived from (duration, window_len, time_step): frame count and centers,
                          centered on the signal midpoint — never on a viewport */ }
impl FrameGrid { pub fn new(duration: f64, window: f64, step: f64) -> Self;
                 pub fn centers(&self) -> impl Iterator<Item = f64>; }
pub struct RealFftPlan { /* cached realfft plans by size */ }
pub fn preemphasis_in_place(x: &mut [f64], from_hz: f64, sample_rate: f64);
pub fn sinc_interpolate_max(y: &[f64], around: usize, depth: usize) -> (f64, f64); // (pos, value)
pub fn next_pow2(n: usize) -> usize;
```
**Constraints.** Standing set. FrameGrid formula: n = floor((duration −
window)/step) + 1, first center at t0 + (duration − (n−1)·step)/2 — document
it; every analysis crate must use this grid (zoom-independence depends on
it). Pre-emphasis per Markel & Gray 1976: y[t] = x[t] − a·x[t−1], a =
exp(−2π·f/fs) (Praat manual "Sound: Pre-emphasize"). Sinc peak interpolation
per Boersma 1993 eq. 22 (citation in
`../../research/algorithms-and-validation.md` §1.2).
**Verification.** Window symmetry/energy tests; FrameGrid property tests
(shifting a query window never changes frame centers); sinc interpolation
recovers the peak of a sampled sinusoid to < 0.01 sample; FFT of known
signals matches analytic magnitudes to 1e-9.

### T1.3 · phx-spectrogram
**Objective.** Gaussian-window STFT power spectrogram with tile API.
**Files.** `crates/phx-spectrogram/src/*`.
**Interfaces.**
```rust
pub struct SpectrogramParams { pub window_length: f64 /* 0.005 */, pub max_frequency: f64 /* 5000 */,
  pub time_step: f64 /* 0.002 */, pub frequency_step: f64 /* 20.0 */, pub window: Window /* Gaussian */ }
pub struct TileRequest { pub t0: f64, pub t1: f64, pub f0: f64, pub f1: f64,
  pub width_px: u32, pub height_px: u32, pub params: SpectrogramParams }
pub struct Tile { pub db: Vec<f32> /* row-major, height×width */, pub t_axis: …, pub f_axis: … }
pub fn compute_tile(audio: AudioView, req: &TileRequest) -> Tile;
pub fn spectral_slice(audio: AudioView, at: f64, params: &SpectrogramParams) -> Slice;
```
**Constraints.** Standing set. PSD in Pa²/Hz → dB; Gaussian −3 dB bandwidth =
1.2982804/windowLength; step clamps time ≥ windowLength/(8√π), frequency ≥
√π/(8·windowLength); display pre-emphasis (+6 dB/oct above 1 kHz:
+6·log2(f/1000) dB) applied as a separate documented function, off in raw
values (open item 1 in the algorithms report). Tiles snap t0/f0 to the
object-level FrameGrid so adjacent tiles share columns exactly.
**Verification.** Pure 1 kHz sine renders a single clean horizontal line
(no sidelobes above −60 dB); overlapping tile requests agree bit-for-bit on
shared columns; magnitude comparison against a scipy STFT reference script
(tools/oracle can host it) within 1e-6 relative.

### T1.4 · phx-render
**Objective.** dB→RGBA tile colorization.
**Files.** `crates/phx-render/src/*`.
**Interfaces.** `pub enum Colormap { Viridis, Magma, Grayscale }`,
`pub struct DisplayMapping { pub dynamic_range_db: f64 /* 50 */, pub max_db: Option<f64> /* autoscale */ }`,
`pub fn colorize(tile_db: &[f32], w: u32, h: u32, map: &DisplayMapping, cm: Colormap, theme: Theme) -> Vec<u8>`.
**Constraints.** Standing set. Colormap tables computed from the published
viridis/magma control points (matplotlib data files are permissively
licensed — cite); grayscale runs white→black on light theme and is re-tuned
(never merely inverted) for dark; linear-in-dB mapping, clip to
[max − dynamicRange, max].
**Verification.** Golden-image tests for each (colormap × theme); monotonic
luminance property test for viridis/magma.

### T1.5 · phx-engine + phx-wasm (walking-skeleton surface)
**Objective.** Minimal engine and WASM bindings: import, waveform pyramid,
tiles.
**Files.** `crates/phx-engine/src/*`, `crates/phx-wasm/src/*`.
**Interfaces.** Engine: `import_wav_bytes(bytes) -> AudioId`,
`waveform_slice(id, t0, t1, px) -> Vec<MinMax>` (from a cached mip pyramid),
`spectrogram_tile_rgba(id, req, display, colormap, theme) -> Vec<u8>`,
`audio_info(id) -> AudioInfo`. WASM: the same functions via wasm-bindgen,
buffers as `js_sys::Uint8Array`/`Float32Array` without extra copies where
possible.
**Constraints.** Standing set; engine functions are stateless-by-arguments
apart from the audio store; no unwrap on user input paths.
**Verification.** wasm-pack test (node) covering import→tile; pyramid
correctness against direct min/max; `cargo build --target
wasm32-unknown-unknown` with simd128.
