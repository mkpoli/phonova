Reviewing model: gpt-5.6-sol, via the Codex CLI. Review date: 2026-07-17.
Verification pass against the repository at commit `9cf2f6a` (2026-07-17).

The review covers five areas: core architecture, oracle/numeric
methodology, `docs/DESIGN.md` against the shipped UI, phase-8 code health,
and pre-publicity gaps (branding, licensing, contributor docs). Each
critical or important finding below was checked against the referenced

and confirmed present unless noted otherwise. Minor findings were
sanity-checked but not re-derived line by line.

## Accepted action items

- Route WAV import, streamed-source open, and recording completion through
  journaled commands so those operations get real undo/replay semantics.
- Wire `oracle-bridge` into CI so the Rust algorithms, not just Parselmouth
  regenerating its own references, are what gets checked.
- Give gross pitch errors, missing formant points, and null-vs-null
  intensity/voice mismatches real failure budgets in `diff.py` instead of
  routing them to notes/counters the pass/fail decision ignores.
- Add case identity, input audio, parameter, timestamp, and span checks to
  the oracle comparator before it compares measurements.
- Fix the delete/undo toast to target its own journal entry instead of
  calling global `undo()`.
- Set an `onerror`/`onmessageerror` handler on the wasm worker and reject
  pending calls on worker failure or `destroy()`.
- Redact the hardcoded home directory and session paths in
  `tools/session-viewer/` and decide whether the tool stays in the tree at
  all.
- State the dual license and link the license files from the README; add
  `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, and `SECURITY.md`; add an
  alpha-status and accuracy disclosure alongside the Praat-replacement
  claim.
- Reconcile the Praat version cited in `gates.md` against `horizon.md`, and
  create the missing `docs/research/upstream-praat.md` log.
